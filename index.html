3. 개인정보의 보유 및 이용 기간

본 앱은 사용자가 서비스를 이용하는 동안 개인정보를 보유 및 이용합니다.

  • 사용자 정보 및 비상연락처: 사용자가 앱을 삭제하거나 정보 삭제를 요청할 때까지 보관
  • 위치 정보: 최근 위치 정보만 저장되며, 새로운 위치 정보로 자동 갱신
  • 활동 기록: 최근 활동 기록만 저장되며, 새로운 활동으로 자동 갱신
  • 로그 파일: 기기 내부에만 저장되며, 7일 후 자동 삭제

사용자가 앱을 삭제하거나 계정 삭제를 요청하는 경우, 관련 개인정보는 즉시 파기됩니다.

4. 개인정보의 제3자 제공

본 앱은 원칙적으로 사용자의 개인정보를 외부에 제공하지 않습니다. 다만, 다음의 경우에는 예외로 합니다.

제공하는 경우

1. 비상 상황 발생 시

  • 제공받는 자: 사용자가 사전에 설정한 비상연락처 1~3명
  • 제공 항목: 사용자 이름, 전화번호, GPS 위치 정보
  • 제공 방법: SMS 문자 메시지
  • 제공 시점: 사용자가 설정한 비활동 시간(12시간/24시간/48시간) 초과 시 서버에서 자동 발송
  • 제공 목적: 비상 상황에서 신속한 안전 확인 및 구조 지원
  • 보유 기간: SMS 발송 후 별도 저장하지 않음 (통신사 보관 정책 따름)

예시 SMS 내용:

[나홀로119 비상알림] 홍길동(010-1234-5678)님이 24시간 동안 활동이 없습니다. 마지막 위치: 서울시 강남구 테헤란로 123 확인 부탁드립니다.

법령의 규정에 의거하거나, 수사 목적으로 법령에 정해진 절차와 방법에 따라 수사기관의 요구가 있는 경우

참고: 비상연락처로 전송된 정보는 수신자의 기기에 저장되며, 본 앱은 전송 이후 해당 정보를 별도로 보관하지 않습니다.

9. 개인정보의 안전성 확보 조치

기술적 조치

1. 암호화 전송

모든 개인정보는 HTTPS(TLS 1.2 이상) 프로토콜을 통해 암호화되어 전송됩니다.

2. 데이터베이스 접근 제어

  • Supabase Row Level Security (RLS) 정책을 통해 기기별 데이터 접근 제한
  • 각 사용자는 자신의 device_id로 등록된 데이터만 조회/수정 가능

3. 로그 관리

  • 앱 내부 로그는 기기 내부 저장소에만 보관
  • 7일 후 자동 삭제
  • 외부로 전송되지 않음

4. 보안 업데이트

정기적인 보안 패치 및 업데이트 적용

관리적 조치

  • 개인정보 처리 시스템 접근 기록 관리: 데이터베이스 접근 로그 보관 및 정기 점검

물리적 조치

  • 서버 보안: Supabase의 물리적 보안 시설 활용
  • AWS 인프라의 물리적 보안 정책 준수

13. 접근 권한 안내

본 앱은 서비스 제공을 위해 다음과 같은 접근 권한을 요청합니다.

필수 접근 권한

1. 위치 정보 (GPS)

Android 권한: ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION, ACCESS_BACKGROUND_LOCATION

사용 목적: 비상 상황 발생 시 사용자의 현재 위치를 비상연락처에게 SMS로 전송

수집 시점:

  • 앱 설정 저장 시
  • 15분마다 백그라운드 체크인 시

저장 방식: 최근 위치만 저장 (이전 위치는 자동 갱신)

거부 시 영향: 비상 시 위치 정보를 전송할 수 없어 서비스 이용 불가

2. 접근성 서비스 (Accessibility Service)

Android 권한: BIND_ACCESSIBILITY_SERVICE

사용 목적:

  • 사용자의 화면 터치 활동 감지
  • 15분간 터치가 없을 경우 비활동 상태로 판단

수집 정보:

  • 마지막 터치 발생 시간만 기록
  • 어떤 앱을 사용했는지, 무엇을 입력했는지는 수집하지 않음
  • 화면 내용, 텍스트, 비밀번호 등은 일체 수집하지 않음
중요: 본 권한은 터치 발생 여부만 확인하며, 개인정보는 수집하지 않습니다

거부 시 영향: 활동 감지가 불가능하여 서비스 이용 불가

3. 알림 권한 (POST_NOTIFICATIONS)

Android 권한: POST_NOTIFICATIONS (Android 13 이상)

사용 목적:

  • 앱 상태 알림
  • 설정 저장 완료 알림
  • 백그라운드 모니터링 상태 알림

거부 시 영향: 알림을 받을 수 없으나 핵심 기능은 정상 작동

4. 배터리 최적화 해제

Android 권한: REQUEST_IGNORE_BATTERY_OPTIMIZATIONS

사용 목적:

  • 백그라운드에서 안정적으로 15분마다 체크인 수행
  • Android의 배터리 절약 모드에서도 서비스 유지

거부 시 영향:

  • 백그라운드 작동이 불안정해질 수 있음
  • 일부 기기에서 체크인이 누락될 수 있음
